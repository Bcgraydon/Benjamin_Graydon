Medium Business Network — Cisco Packet Tracer

Layer 3 core (SVIs) + segmented department VLANs + centralized services + ACL-driven access control.

Overview Topology VLANs Services Security Policy Evidence Validation Download

Quick Facts

  • Routing: L3 core switch with SVIs + ip routing
  • Segmentation: 9 VLANs (departments + servers + printers + guest + mgmt)
  • Services VLAN: DHCP, DNS, Intranet (HTTP), File server
  • DHCP delivery: DHCP relay via ip helper-address on SVIs
  • Security: ACLs enforce “deny-by-default” inter-VLAN policy with approved exceptions
  • Guest: Internet/DHCP allowed; internal networks blocked

Overview

This Packet Tracer project models a medium-sized business network with department VLANs, a Layer 3 core switch providing inter-VLAN routing via SVIs, and centralized services hosted in a dedicated Servers VLAN. ACLs enforce segmentation with controlled cross-access between approved departments and restricted access to the Management VLAN.

Topology

  • Core: Layer 3 switch (SVIs + ip routing)
  • Access: Three access switches (802.1Q trunk uplinks, access ports per department)
  • Edge: Router uplink for default route (optional internet simulation)
  • Services: Dedicated Servers VLAN with DHCP, DNS, intranet web, and file server

Access Control and Segmentation

Inter-VLAN access is restricted by default and enabled only where business requirements exist. Guest traffic is isolated from internal networks while still receiving DHCP via relay. Management IPs are restricted to IT.
