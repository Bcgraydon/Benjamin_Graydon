Medium Business Network - Packet Tracer Project

Overview

This Packet Tracer project models a medium-sized business network with multiple departments, each on its own VLAN. Inter-VLAN routing is handled by a Layer 3 core switch using SVIs. Centralized services (DHCP, DNS, web, file) are hosted on separate servers, and ACLs enforce segmentation with controlled cross-access between specific departments.

Topology

  • Core: Layer 3 switch (SVIs + ip routing)
  • Access: 3 access switches (trunk uplinks, access ports per department)
  • Edge: Router uplink for default route (optional internet simulation)
  • Services: Separate DHCP, DNS, Web, and File servers

VLANs and Addressing

  • VLAN 10 - HR - 10.10.10.0/24 (GW 10.10.10.1)
  • VLAN 20 - Finance - 10.10.20.0/24 (GW 10.10.20.1)
  • VLAN 30 - Sales - 10.10.30.0/24 (GW 10.10.30.1)
  • VLAN 40 - IT - 10.10.40.0/24 (GW 10.10.40.1)
  • VLAN 50 - Operations - 10.10.50.0/24 (GW 10.10.50.1)
  • VLAN 60 - Servers - 10.10.60.0/24 (GW 10.10.60.1)
  • VLAN 70 - Printers - 10.10.70.0/24 (GW 10.10.70.1)
  • VLAN 80 - Guest - 10.10.80.0/24 (GW 10.10.80.1)
  • VLAN 99 - Management - 10.10.99.0/24 (GW 10.10.99.1)

Services

  • DHCP Server: 10.10.60.10 (DHCP relay via ip helper-address on SVIs)
  • DNS Server: 10.10.60.11
  • Web Server (Intranet): 10.10.60.12
  • File Server: 10.10.60.13

Access Control and Segmentation

  • Guest VLAN: blocked from internal VLANs; DHCP works via relay
  • Management VLAN: only IT VLAN can access management IPs (ACL enforced on VLAN 99 outbound)
  • Cross-access: HR ↔ Finance allowed; Sales ↔ Operations allowed
  • Default segmentation: other inter-department access is restricted

Validation

  • DHCP leases issued correctly per VLAN (correct IP, gateway, DNS)
  • DNS resolution to intranet hostname and successful HTTP access
  • Guest blocked from internal resources while still receiving DHCP
  • HR cannot reach Sales; HR can reach Finance
  • Sales can reach Operations; other cross-department access restricted
  • Only IT can access management VLAN IPs
