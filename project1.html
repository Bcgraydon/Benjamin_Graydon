Project 1 — Event-Driven Programming (Nile Dot Com)

Description

A standalone Java GUI that simulates an e-store (“Nile Dot Com”). Users search inventory, add up to five distinct items to a shopping cart, see a running subtotal, then check out to generate an invoice and append a transaction record. The app reads an input file inventory.csv (id, quoted description, in-stock status, quantity, unit price) and appends order details to transactions.csv with a unique timestamp-based id.

The main window includes labeled input areas, a live cart display, a running subtotal, and six workflow buttons that enable/disable as the user progresses (search, add to cart, delete last item, check out, clear/new order, exit).

Constraints and pricing logic: cart capacity is limited to 5 items; tax rate is 6%; volume discounts apply per-item: 1–4 → 0%, 5–9 → 10%, 10–14 → 15%, 15+ → 20%.

Error handling follows the spec with dialogs for out-of-stock, item-not-found, and quantity-exceeds-stock; checkout shows an invoice and writes to the transaction log.
