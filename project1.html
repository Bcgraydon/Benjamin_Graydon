Project 1 — Event-Driven Programming (Nile Dot Com)

Description

A standalone Java GUI that simulates an e-store (“Nile Dot Com”). Users search inventory, add up to five distinct items to a shopping cart, see a running subtotal, then check out to generate an invoice and append a transaction record. The app reads inventory.csv (id, quoted description, in-stock status, quantity, unit price) and appends order details to transactions.csv with a timestamp-based id.

The main window includes labeled input areas, a live cart display, a running subtotal, and six workflow buttons that enable/disable as the user progresses (search, add to cart, delete last item, check out, clear/new order, exit).

Pricing and constraints: cart capacity 5 items; tax 6%; tiered discounts per item — 1–4 (0%), 5–9 (10%), 10–14 (15%), 15+ (20%).

Skills Learned

  • Event-driven programming with Java GUIs (listeners, state-driven UI)
  • CSV file I/O for inventory and transaction logs
  • Tax/discount calculations and currency formatting
  • Error dialogs and UX flow control (enable/disable actions)
