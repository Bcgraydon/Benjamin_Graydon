Project 2 — Synchronized/Cooperating Threads: Banking Simulator

Description

A Java multithreaded simulation of a bank with two shared accounts and five agent types running concurrently: depositors, withdrawers, transfer agents, an internal auditor, and a U.S. Treasury auditor. Access is synchronized with reentrant locks and conditions; withdrawers block when funds are insufficient and are signaled after deposits. Transfers and audits acquire multiple locks together for atomic operations; if locks aren’t available, agents back off and retry.

The program uses a fixed thread pool to launch agents, randomized pacing to vary event timing, and ordered console/log output for readability and auditing snapshots.

Skills Learned

  • Designing cooperating, synchronized threads (producer/consumer patterns)
  • Using ReentrantLock, conditions (await/signal), and tryLock
  • Atomic multi-resource operations with lock ordering (transfers, audit snapshots)
  • ExecutorService thread pools, pacing/jitter, and structured logging
