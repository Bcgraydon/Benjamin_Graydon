Project 2 — Synchronized/Cooperating Threads: Banking Simulator

Description

A Java multithreaded simulation of a bank with two shared accounts and five agent types running concurrently: depositors, withdrawers, transfer agents, an internal auditor, and a U.S. Treasury auditor. Access to the shared accounts is synchronized using java.util.concurrent locks to ensure mutual exclusion, with withdrawing threads blocking when funds are insufficient and being awakened after deposits. Transfers and audits are atomic operations that require holding multiple account locks together; if all locks can’t be acquired immediately, those agents release what they hold and try again later. The simulation emphasizes proper coordination, random timing of events, and consistent snapshotting during audits.

Skills Learned

  • Designing cooperating, synchronized threads (producer/consumer–style patterns).
  • Using reentrant locks, conditions (await/signal), and non-blocking tryLock to avoid deadlock.
  • Coordinating atomic multi-resource operations (transfers, auditable snapshots) with lock ordering.
  • ExecutorService with a fixed thread pool to manage agent lifecycles and pacing.
  • Defining clear thread responsibilities and state machines for realistic simulations.
