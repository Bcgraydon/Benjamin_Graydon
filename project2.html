Project 2 - Synchronized/Cooperating Threads: Banking Simulator

Description

A Java multithreaded simulation of a bank with two shared accounts and multiple agent types running concurrently: depositors, withdrawers, transfer agents, an internal auditor, and a U.S. Treasury auditor.

Access is synchronized with reentrant locks and conditions. Withdrawers block when funds are insufficient and are signaled after deposits. Transfers and audits acquire multiple locks together for atomic operations, and agents back off and retry if locks are unavailable.

The program uses a fixed thread pool to launch agents, randomized pacing to vary event timing, and ordered console or log output for readability and auditing snapshots.

Skills Learned

  • Designing cooperating, synchronized threads
  • Using ReentrantLock, conditions, and tryLock
  • Atomic multi-resource operations with lock ordering
  • ExecutorService thread pools and structured logging
